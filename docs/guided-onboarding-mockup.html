4 Explain + confirm

Each install is its own confirmable step.

The user reads what's about to happen, sees the exact command, and clicks Continue. "Skip — I already have this" lets devs short-circuit if our detection missed their install. Three confirms for a fresh Codex install: Homebrew → Node.js → Codex CLI. Each one auto-skips if already detected, so a dev with brew + node already installed only sees the Codex CLI confirm. Gemini follows the same three-confirm pattern with @google/gemini-cli. Copilot is two confirms (Homebrew + gh) — no Node needed.

Install Homebrew

Tokenomics needs Homebrew to install Node.js and the Codex CLI. Homebrew is the standard Mac package manager — about 2 minutes to install.

Tokenomics will run:
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

We'll open Terminal and install Homebrew with your permission. You'll be asked for your password once. Same official installer from brew.sh.

Install Node.js

Tokenomics needs Node.js (and npm, which comes with it) to install the Codex CLI. About 30 seconds.

9 Full sequence

A fresh-Mac Codex setup, end-to-end.

The path a brand-new user walks through. Each frame is one screen above — this just makes the cadence visible.

  1. Welcome → "Get started"
  2. Pick provider → tap OpenAI
  3. Detect → "Homebrew, Node, Codex CLI all missing" → "Continue"
  4. Confirm install Homebrew → "Install Homebrew"
  5. Progress → live log → completes
  6. Confirm install Node → "Install Node.js"
  7. Progress → live log → completes
  8. Confirm install Codex CLI → "Install Codex CLI"
  9. Progress → live log → completes
  10. Auth handoff → "Open browser to sign in"
  11. (user signs in in browser)
  12. Done → "Add another or I'm all set"

A dev who already has Homebrew and Node skips steps 4–7 and lands directly at step 8. Same screens, same vocabulary.

Provider variations.

The canonical flow above (Codex / Pattern A) covers Tokenomics-driven install + automated browser OAuth. Three other patterns diverge enough to warrant their own screens — Cursor doesn't get installed by us, API-key providers don't install at all, and Claude's auth has to be driven by the user (Anthropic policy). Each one reuses the same window chrome and stepper vocabulary, just with different content and labels.

10 Cursor — app-bundle flow

Pattern D: download an app, sign into it, wait for both.

Tokenomics doesn't install Cursor — Cursor is a real Mac app distributed from cursor.com. The "Installing tools" step renames to "Installing Cursor" and becomes a hand-off + poll instead of a streaming install. Tokenomics needs both the app bundle and the user's sign-in (the JWT only gets written to Cursor's local config once they've signed in once). The polling state watches for both.

11 API key — paste flow

Pattern E: get a key from the provider, paste it here.

No install, no OAuth — these providers expose a long-lived API key that Tokenomics reads directly. Steps rename to fit: "Installing tools" becomes "Get API key", "Signing in" becomes "Paste key". The key lives in macOS Keychain after this; Tokenomics never holds it in memory or sends it anywhere except the provider's own usage endpoint.

12 Anthropic — user-driven sign-in

Pattern B: we start it, you finish it.

Anthropic now ships Claude Code via Homebrew (brew install --cask claude-code), which sidesteps the npm prefix and PATH issues. Sign-in itself is still multi-step inside Claude Code — Anthropic's policy doesn't let third-party apps drive it. So Tokenomics frames Step 2 (Installing tools) as a handoff: we install the CLI, then walk the user through what they'll need to do in Terminal to finish setup. Five windows total — two confirm screens for the actual installs (Homebrew, then Claude Code), two preview screens that prep the user for the Claude Code wizard, then a waiting screen with the "you're done when you see this" cue. Step 3 (Signing in) activates the moment Terminal opens. Any window 1–4 auto-skips if its condition is already satisfied — so a dev with brew + claude-code + auth already in place would land directly on the Done frame.
